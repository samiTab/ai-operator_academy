## 0. Why this can win (the strategic wedge)

Research into the current market (Coursiv, Anthropic Academy, top-selling 2026 AI courses) surfaced three facts that dictate the design:

1. **Coursiv's formula works but is shallow.** Daily 5–10 min micro-lessons, "explain → practice → quiz → streak," beginner-friendly, certificates, in-app playground. 800k+ learners, 4.4★. Reviewers' #1 complaint: *no depth for advanced users and no real personalization*. That gap is our wedge.
2. **Anthropic runs a FREE official "Anthropic Academy"** (17 courses on Claude, Claude Code, Cowork, Skills, subagents, MCP — certificates included). **We cannot win by re-teaching features.** A paid product must win on *adaptivity, business outcomes, done-with-you implementation, and speed-to-ROI*.
3. **The best-selling 2026 courses** all share: hands-on > theory, role-specific paths, verifiable certificates, explicit ROI framing, and "living curriculum" that updates as tools change.

**Our positioning in one line:** *Not another "learn the buttons" course. An adaptive system that turns YOUR actual work into automated workflows and AI teammates — and proves the ROI.*

> ⚠️ **Naming/trademark note.** "Claude" and "Claude Code" are Anthropic trademarks. Using them *descriptively* ("Master Claude AI & Claude Code") is normal and fine. Naming the **product** "Claude [X]" risks implying official endorsement. Recommendation: keep a neutral brand ("AI Operator Academy") + descriptive subtitle. Final call is yours.

## 2. The adaptive engine (the core IP)

This is what justifies the price and beats free alternatives. Personalization happens on **four axes**:

| Axis | What adapts | Driven by |
|---|---|---|
| **Content** | Which modules/lessons appear, which are skipped | Role, goal, current tool stack |
| **Flow / order** | Sequence and pacing of lessons | Purpose ("automate invoicing now" vs "broad literacy") |
| **Depth** | Conceptual vs. just-do-it; how much "why" | Self-rated experience + early performance |
| **Difficulty** | Exercise complexity, hand-holding level | Live performance signals (see below) |
| **Examples / case studies** | Industry- and function-specific scenarios | Industry + function from enrollment |

### 2.1 Enrollment diagnostic (the ~8 questions)

Short, tappable, mobile-friendly. Captures the variables the personalizer needs. Recommended set:

1. **Who are you?** Business owner / Manager / Independent (freelancer/solo) / Team lead
2. **Industry?** (Marketing, Real estate, Legal, E-commerce, Consulting, Healthcare admin, Finance/Accounting, Education, Trades/Services, Other → free text)
3. **Primary goal right now?** Automate a repetitive workflow / Build a tool or solution / Boost my own output / Train my team / Explore what's possible
4. **The #1 task you'd love to hand off?** (free text — this becomes their capstone seed)
5. **Experience with AI tools?** Never used / Use ChatGPT-style chat sometimes / Use AI daily / I've built things with AI
6. **Comfort with technical tools?** (terminal, files, code) Avoid it / Willing if guided / Comfortable / I code
7. **Time per week?** <1h / 1–3h / 3–5h / 5h+
8. **What does success look like in 30 days?** (free text — defines the "north-star outcome")

> Two free-text answers (Q4, Q8) are the gold. They let the system generate a *truly* personalized capstone and ROI target rather than a generic track.

### 2.2 The four base tracks (composed, not fixed)

The personalizer composes a path from a module library. Most learners are a *blend*, weighted by their answers:

- **Track A — The Operator** (own output): prompting mastery, Claude as daily co-worker, document/data/research workflows, personal automations.
- **Track B — The Builder** (solutions): Claude Code basics for non-coders, building small tools/apps, Skills, MCP connectors.
- **Track C — The Automator** (workflows): mapping a workflow, Cowork task loops, agents/subagents, scheduled & multi-step automations.
- **Track D — The Team Leader** (AI as teammate): designing "AI roles," SOPs + Skills for a team, governance, rollout, measuring impact.

Difficulty/depth then scales each chosen module up or down.

### 2.3 Live difficulty signals (how the course "watches and adjusts")

The `progress-coach` skill reads these signals and re-tunes the next lesson:

- Exercise pass on first try → bump difficulty / reduce scaffolding.
- 2+ retries or asks for hint → add a worked example, slow down, drop optional depth.
- Time-per-lesson far above estimate → shorten upcoming lessons, surface a "fast path."
- Skips theory consistently → favor just-do-it framing.
- Strong performance + "I code" → unlock advanced/bonus tracks (raw API, advanced MCP, eval harnesses).

## 3. Lesson anatomy (the repeatable unit)

Borrowed from Coursiv's proven loop, upgraded with a **real sandbox** and **ROI capture**:

```
1. HOOK (15s)        — why this matters to YOU, in your industry's words
2. CONCEPT (2–4 min) — minimal "why," depth scaled to the learner
3. DEMO              — annotated screenshots / short screen-capture of the exact steps
4. DO IT (the core)  — guided hands-on task in a live Claude sandbox (or their own account)
5. CHECK             — auto-graded by exercise-grader, with specific feedback, not pass/fail
6. APPLY             — "now do it with YOUR real task" prompt (ties to their Q4/Q8 answers)
7. RETAIN            — 1 micro-quiz Q + a streak/XP nudge
8. ROI BEAT          — "you just saved ~X min/week" tally feeding the certificate + dashboard
```

Target: **7–12 min** per lesson so it fits the busy-professional habit, but depth-adaptive (advanced learners get denser, longer optional segments).

## 4. Curriculum map (module library)

Modules are **composable**. A learner sees a subset, reordered and re-leveled. Numbers are reference IDs, not a fixed sequence.

### Foundations (almost everyone)
- **F1. The mindset shift** — AI as a teammate, not a search box. What Claude is good/bad at (honest capabilities & limits). Case study format.
- **F2. Talking to Claude that gets results** — practical prompting: context, examples, format, iteration. (Role-flavored examples.)
- **F3. Your first real win** — pick one painful task → solve it with Claude in 10 min (uses their Q4 answer).

### Track A — Operator (own output)
- **A1. Claude as your chief of staff** — email, summaries, planning, decisions.
- **A2. Documents & data** — analyze, draft, reformat, extract; spreadsheets and reports.
- **A3. Research & synthesis** — fast, sourced research; competitive/market scans.
- **A4. Your personal prompt & Skills library** — reusable assets so you stop re-typing.

### Track B — Builder (solutions, non-coder-friendly)
- **B1. Claude Code without fear** — what it is, install/setup, the Explore→Plan→Code→Commit loop.
- **B2. Build a small internal tool** — e.g., a quote generator, a data cleaner, a mini dashboard.
- **B3. Agent Skills** — package your know-how into reusable Skills Claude applies automatically.
- **B4. Connect your tools (MCP)** — wire Claude to the apps you already use; when to use connectors.

### Track C — Automator (workflows)
- **C1. Map a workflow worth automating** — choose, scope, measure baseline time/cost.
- **C2. Claude Cowork task loops** — delegate real multi-step desktop/knowledge work.
- **C3. Agents & subagents** — break big jobs into delegated, parallel tasks.
- **C4. Scheduled & multi-step automations** — make it run without you; handle errors & review gates.

### Track D — Team Leader (AI as teammate)
- **D1. Design an "AI role"** — write a job description for an AI teammate; scope its lane.
- **D2. Team Skills & SOPs** — turn your playbooks into shared Skills; consistency at scale.
- **D3. Rollout & adoption** — change management, training your people, avoiding the common failures.
- **D4. Governance, safety & measurement** — data handling, review checkpoints, tracking ROI honestly.

### Capstone (everyone)
- **CAP. Ship your asset** — guided build of the thing from their enrollment answers. Output is portfolio-ready and demoed back. Unlocks the certificate.

> Each module ships with: ≥1 **case study**, ≥1 **industry-flavored example set**, a **template/asset** to keep, and a **common-mistakes** callout. The `lesson-author` skill enforces this.

## 5. Certificate & credential

- **Verifiable completion certificate** with a unique ID + public verification URL (combats the "is this real?" doubt that hurts competitors).
- Earned by completing the path **and** shipping the capstone (not just watching).
- Shows: track(s) completed, the **capstone asset built**, and **measured ROI** (time saved/week). This makes it a portfolio piece, not just a badge.
- LinkedIn-shareable. Honest framing: "demonstrates practical, hands-on capability" — not academic accreditation.

## 6. Platform architecture (how it's actually built)

The platform is a thin app wrapped around a **Claude-powered "academy brain."** The brain is a set of **Agent Skills** (in `/skills`) coordinated by orchestration logic (in `/agents`). This is the part the brief asked to build first.

```
┌──────────────────────────────────────────────────────────────┐
│  LEARNER APP  (web + mobile, e.g. Next.js + React)             │
│  enrollment · lesson player · sandbox · dashboard · cert       │
└───────────────┬──────────────────────────────────────────────┘
                │  calls
┌───────────────▼──────────────────────────────────────────────┐
│  ACADEMY BRAIN  (Claude via API, orchestrating Agent Skills)   │
│                                                                │
│  • course-personalizer   → builds & re-tunes each path         │
│  • lesson-author         → generates adaptive lesson content   │
│  • exercise-grader       → evaluates practice, gives feedback  │
│  • progress-coach        → reads signals, adjusts difficulty   │
│  • certificate-issuer    → validates completion, issues cert   │
│  • marketing-generator   → produces launch & growth assets     │
└───────────────┬──────────────────────────────────────────────┘
                │  reads/writes
┌───────────────▼──────────────────────────────────────────────┐
│  DATA  learner profile · progress/signals · asset library ·    │
│        ROI ledger · certificate registry                       │
└────────────────────────────────────────────────────────────────┘
```

**Build options (pick per budget/timeline):**
- **Fast path (validate first):** an existing LMS (Teachable/LearnWorlds/Thinkific) for hosting + payments + certificates, with the adaptive brain bolted on via the Claude API and an embedded sandbox. Ship in weeks.
- **Custom path (the real product):** Next.js app + Claude API + a small DB (the architecture above). Full control over the adaptive experience and the in-lesson sandbox. This is where the moat lives.

**The in-lesson sandbox** is critical (it's Coursiv's "AI Playground," upgraded): learners practice against a real Claude instance inside the lesson, with the exercise pre-loaded, so there's zero setup friction and the grader can see their work.

## 7. Business model (informed by the market)

- **Pricing posture:** avoid the trust-eroding "hidden weekly subscription" trap that dominates Coursiv complaints. Be transparent.
- **Recommended structure:**
  - **Core (one-time or clear monthly):** full adaptive course + certificate + asset library.
  - **Pro/Cohort (higher tier):** live office hours, reviewed capstone, team Skills library, updates.
  - **Team/Enterprise:** seats + admin dashboard + custom industry case studies + rollout support.
- **"Living curriculum"** is a paid-tier feature: when Claude ships new capabilities, new lessons appear. The `marketing-generator` + `lesson-author` skills make this cheap to maintain.
- **ROI guarantee** angle: "ship one automation that saves you 2+ hrs/week or your money back" — credible because the capstone is built in.
